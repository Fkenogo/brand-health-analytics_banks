## 1.2 Competitive Gap Analysis

**Calculate performance gaps:**

```
For each metric:
    Gap = Your Bank - Competitor
    Gap % = (Gap ÷ Competitor) × 100

Example:
Your NPS: +25
Leader NPS: +40
Gap: -15 points
Gap %: -15 ÷ 40 = -37.5% (You're 37.5% below leader)
```

**Categorize gaps:**
```
Large Advantage: Your bank >20% better
Small Advantage: 5-20% better
Parity: ±5%
Small Disadvantage: 5-20% worse
Large Disadvantage: >20% worse
```

# 2. COMPETITIVE INTELLIGENCE

## Overview
Deeper analysis of competitive dynamics, market positioning, and strategic insights.

---

## 2.1 Market Share Estimation

**Based on "Currently Using" data:**

```
Total Sample: 1000 people
Bank A current users: 220 → 22% market share
Bank B current users: 180 → 18%
Bank C current users: 150 → 15%
Bank D current users: 120 → 12%
Others: 330 → 33%
────────────────────────────────
Total: 1000 → 100%
```

**Market Concentration:**
```
HHI (Herfindahl Index) = Sum of squared market shares
HHI = 22² + 18² + 15² + 12² + 33² = 2342

IF HHI < 1500: Competitive market
IF HHI 1500-2500: Moderate concentration
IF HHI > 2500: Highly concentrated
```

---

## 2.2 Share of Wallet Analysis

**Multi-banking behavior:**

```
Average Banks per Customer = Total active accounts ÷ Banking population

Bank A Average: Customers use avg 2.3 banks
Bank B Average: Customers use avg 1.8 banks

→ Bank A has more multi-bankers (lower loyalty or better for specific needs)
→ Bank B has more exclusive users (higher loyalty)
```

**Wallet Penetration:**
```
For Bank A customers:
- 40% use Bank A only
- 35% use Bank A + 1 other
- 25% use Bank A + 2+ others

Primary Bank Rate = Only users + (Multi-bankers where A is primary) ÷ Total users
```

---

## 2.3 Competitive Set Analysis

**Identify direct competitors:**

```
For your bank's current users:
    Which OTHER banks do they use?
    
Results:
Bank A users also use:
- Bank B: 45% overlap
- Bank C: 32% overlap
- Bank D: 28% overlap

→ Bank B is your primary competitor (competing for same customers)
```

**Switching Patterns:**
```
For lapsed users:
    Where did they go?
    
Of 150 lapsed Bank A users:
- 60 switched to Bank B (40%)
- 35 switched to Bank C (23%)
- 55 unknown/multiple (37%)

→ Bank B is capturing your churn
```

---

## 2.4 Win/Loss Matrix

**Track customer movement between banks:**

```
                 Gained From                 Lost To
        Bank A  Bank B  Bank C  |  Bank A  Bank B  Bank C
Bank A    --      30      20    |   --      40      25
Bank B    40      --      15    |   30      --      20
Bank C    25      20      --    |   20      15      --

Net Position:
Bank A: Gained 50, Lost 65 → Net -15 (losing customers)
Bank B: Gained 55, Lost 50 → Net +5 (gaining customers)
Bank C: Gained 45, Lost 40 → Net +5 (gaining customers)
```

**Churn Source Analysis:**
```
Bank A losing most to: Bank B (40 customers)
Bank A gaining most from: Bank B (30 customers)
Net to Bank B: -10 customers

Recommendation: Study why customers prefer Bank B
```

## 4.5 Growth Rate Calculations

**Calculate CAGR (Compound Annual Growth Rate):**

```
CAGR = (Ending Value ÷ Beginning Value)^(1/years) - 1

Example:
2023: 50% awareness
2026: 75% awareness (3 years)

CAGR = (75 ÷ 50)^(1/3) - 1 = 14.5% annual growth
```

**Monthly Growth Rate:**

```
Monthly Rate = (Current ÷ Previous)^(1/1) - 1

Example:
Jan: 68%
Feb: 70%

Monthly Growth = (70 ÷ 68) - 1 = +2.9%

Annualized: (1 + 0.029)^12 - 1 = 41% annual run-rate
```

## 4.7 Volatility & Stability Metrics

**Standard Deviation:**

```
Monthly awareness values: 68, 70, 69, 71, 73, 75
Mean: 71
StdDev: 2.5

Interpretation:
- StdDev < 3: Stable
- StdDev 3-5: Moderate volatility
- StdDev > 5: High volatility
```

**Coefficient of Variation:**

```
CV = (StdDev ÷ Mean) × 100
CV = (2.5 ÷ 71) × 100 = 3.5%

<10%: Low volatility (predictable)
10-20%: Moderate
>20%: High volatility (unpredictable)
```

# 5. DATA QUALITY & SAMPLE

## Overview
Ensure data reliability, representativeness, and validity.

---

## 5.1 Sample Size Tracking

**Monitor sample over time:**

```
Target: 1000 responses/month

Jan: 987 responses (98.7% of target)
Feb: 1,043 responses (104.3%)
Mar: 921 responses (92.1%) ⚠️ Below target
Apr: 1,102 responses (110.2%)

Average: 1,013/month
Completion: 101% of target
```

**Sample Size Adequacy:**

```
Margin of Error = 1.96 × √(p(1-p)/n)

Where:
p = proportion (e.g., 0.70 for 70% awareness)
n = sample size

Example:
n = 1000, p = 0.70
MOE = 1.96 × √(0.70 × 0.30 ÷ 1000) = ±2.8%

At 95% confidence: True awareness is 70% ± 2.8% (67.2% - 72.8%)
```

---

## 5.2 Response Rate Tracking

**Calculate response metrics:**

```
Survey invitations sent: 5,000
Survey started: 1,500 (30% start rate)
Survey completed: 1,000 (20% completion rate)

Drop-off rate: 1,500 - 1,000 = 500 (33% of starters)

Where drop-offs occur:
- After screening: 200 (40%)
- Mid-survey (Section C): 150 (30%)
- Near end (Section E): 150 (30%)
```

**Completion Time:**

```
Median time: 8 minutes
Mean time: 10 minutes
Range: 3-45 minutes

⚠️ Flag responses <3 min (rushing) or >30 min (distracted)
```

---

## 5.3 Data Quality Checks

**Identify suspicious responses:**

### **1. Speeding:**
```
IF completion time < 50% of median:
    Flag as "rushed response"
    
Example: Survey median = 8 min, flag <4 min
```

### **2. Straight-lining:**
```
IF all rating questions have same answer:
    Flag as "low quality"
    
Example: All NPS questions rated exactly 5/10
```

### **3. Inconsistencies:**
```
IF Currently Using = Yes BUT Ever Used = No:
    Logic error - flag for review
    
IF Preferred Bank ≠ among Currently Using:
    Inconsistency - flag
```

### **4. Missing Data:**
```
Calculate % complete for each respondent:

Respondent A: 100% complete ✓
Respondent B: 85% complete (missing demographics)
Respondent C: 60% complete (abandoned mid-survey) ⚠️
```

---

## 5.5 Weighting Strategy

**If sample is not representative:**

```
Weight Calculation:

Age 18-24 weight = (Population % ÷ Sample %) 
                 = (22% ÷ 18%) = 1.22

Apply weights:
Raw responses from 18-24: Count each as 1.22
Raw responses from 25-34: Count each as 0.88 (28% ÷ 32%)

Weighted total should match population distribution
```

---

## 5.6 Data Reliability Metrics

**Test-Retest Reliability:**

```
IF asking same people twice:
    Calculate correlation between Time 1 and Time 2
    
Good reliability: r > 0.80
Moderate: r = 0.60-0.80
Poor: r < 0.60
```

**Internal Consistency:**

```
IF multiple questions measure same construct:
    Calculate Cronbach's Alpha
    
Good consistency: α > 0.70
Acceptable: α = 0.60-0.70
Poor: α < 0.60
```
